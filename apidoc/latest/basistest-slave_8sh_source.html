40 function print_options
41 {
42  cat - << EOF-OPTIONS
43 Optional arguments:
44  --project, -p The name of the project to be tested.
45  --branch, -b The branch to be tested, e.g., "tags/1.0.0".
46  Default: "trunk".
47  --model, -m The name of the dashboard model, i.e., either "Nightly",
48  "Continuous", or "Experimental".
49  Default: "Experimental".
50  --script, -S CTest script which performs the testing.
51  Default: basistest.ctest script of BASIS.
52  --args, -a Additional arguments for the CTest script. See below.
53  Instead of using the --args option, the additional arguments
54  can be given directly with only two dashes (--) prefixed.
55 
56 Standard arguments:
57  --verbose, -v Increases verbosity of output messages. Can be given multiple times.
58  --help, -h Print help and exit.
59  --helpshort Print short help and exit.
60  --version Print version information and exit.
61 
62 All other arguments are passed on as additional arguments to the CTest script
63 without the leading two dashes (--). For example, the optional argument
64 --coverage corresponds to supplying the option --args with the value 'coverage'.
65 Both result in the argument 'coverage' being passed on to the CTest script.
66 See below for a list of valid arguments of the CTest script.
67 EOF-OPTIONS
68 }
